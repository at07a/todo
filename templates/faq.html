Questions you may be wondering

How is data stored in this project?

Everything is stored in a SQLite database. The Python backend uses SQLAlchemy ORM to make Python classes (BoardList and Task) for their corresponding database tables.

Which programming languages are used in this project?

The backend is written in Python and uses Flask to run. The frontend is written in HTML, CSS, and JavaScript. Admittedly, I had to watch a couple of tutorials on this since some of it was new to me.

What happens with dragging and dropping between lists?

When your mouse hovers over a task, you can click and hold to move it to another list. This sends a request to the backend to update it.

How is the app aware of my lists?

The app creates a unique session-based ID for every visitor. I decided to use this after testing mobile compatibility and discovering that whatever you do on one device affects all others.

What happens when I delete a list?

When you delete a list, it gets removed from the database completely. Because of how the backend is set up, any tasks that were inside that list also get deleted so they don’t just sit there doing nothing.

How do I add a new task?

When you add a task on the frontend, it sends the info to the backend through a POST request. The backend then saves it to the database and sends the updated data back so it appears without you refreshing.

Can I use this on my phone?

No, I tried testing it on my phone and it wouldn’t work. I tried adding compatibility for mobile devices, but I just have no clue how.

Why did you choose SQLite instead of something else?

I went with SQLite because it’s simple to set up and works for small projects like this. I didn’t need something huge like PostgreSQL or MySQL for what I was doing.

Where is the code for the drag-and-drop feature?

The code for the drag-and-drop feature is located in static/app.js. It listens for when you hold your mouse on a card, tracks where your mouse is, and sends the update to the backend.

Does the app save automatically?

Yes, everything you do in the app—such as moving, adding, or deleting tasks—is sent to the backend and saved instantly without you pressing a button.

Can I share my lists with someone else?

Not really. Because it’s based on your session ID, someone else would need to have the exact same session cookie to see your stuff, which basically means using the same device and browser.

What happens if the database file gets deleted?

If the database file got deleted, all tasks made by users and custom lists would be lost, since the backend automatically generates the file with only "To-do", "In Progress", and "Completed".

How long did it take to make?

It took me about 3 days in total. I spent most of the first day watching tutorials on how to properly set this up, because as I coded I kept running into errors I had no clue about, plus a few things I just didn’t know how to do.

What was the hardest part to code?

The hardest part for me was probably getting the changes in the frontend to match up with the backend and making sure they stayed permanent just for the client. That became very confusing as I added more and more features.

Did you have any inspiration?

Yes, I got inspiration for the design and the whole idea from a website called Trello. I use Trello often when working on personal projects.
